# Принятые решения

## 08.11.2025 - Старт проекта
**Решение:** Начинаем с инфраструктуры и минимального MVP
**Обоснование:** Согласно ТЗ Фаза 1 - демонстрация "магии" для HTTP-запросов
**Следствия:** Фокусируемся только на компонентах для HTTP-сценария

## 08.11.2025 - Технологический стек
**Решение:** Java/Spring Boot, React/TS, PostgreSQL, Elasticsearch, OpenTelemetry
**Обоснование:** Соответствует требованиям ТЗ, универсально, богатая экосистема
**Следствия:** Быстрый старт, хорошая поддержка observability

## 08.11.2025 - Архитектура репозиториев  
**Решение:** Отдельные репы для каждого компонента + management репа
**Обоснование:** Профессиональный подход, изоляция, независимая разработка
**Следствия:** Нужно настроить связи между репозиториями
